# Текущая структура презентации

Источник: `slides.md` на момент фиксации. Это рабочая карта текущего хода, её можно менять при перестройке презентации.

## Слайды

1. **Титул** — тема, датасет, фокус: спрос, категории, крупные покупки, прогноз.
2. **01 · Задача и данные** — вопрос исследования, проверки, ограничения датасета, 2017 как основной год.
3. **02 · Покрытие данных по годам** — график покрытия, объяснение выбора 2017 как базового года.
4. **03 · Методология** — seasonal index, CV, peak month/quarter, тип сезонности, rolling z-score, P90, ML-проверка.
5. **04 · Заказы по месяцам, 2017** — общий месячный спрос и рост к Q4.
6. **05 · Детрендированный дневной спрос** — событийные всплески после снятия тренда.
7. **06 · Категории: кто сезоннее** — event-категории, лидеры по сезонности, ограничение SKU-уровня.
8. **07 · Топ категорий по коэффициенту вариации** — категории с сильными месячными колебаниями.
9. **08 · Heatmap: сезонность по месяцам и категориям** — месячные индексы по категориям.
10. **09 · Крупные покупки по месяцам** — high-ticket demand отдельно от массового спроса.
11. **10 · Доля крупных заказов с доверительными интервалами** — устойчивость различий по месяцам.
12. **11 · Какие гипотезы проверяли** — спрос, бизнес-метрики, прогноз.
13. **12 · Результаты проверки гипотез** — что подтвердилось и что не стало универсальным решением.
14. **13 · ML-модель: ROC-кривая** — ML как контрпроверка, не основной механизм.
15. **14 · Можно ли предсказать сезонность заранее?** — рабочий подход через категорийный профиль и календарь событий.
16. **15 · Операционная сезонность** — inventory, sellers, promo, delivery, reviews, freight, недельная нагрузка.
17. **16 · Спрос по дням недели, 2017** — недельный профиль для операционного планирования.
18. **17 · Выводы и ограничения** — неоднородность сезонности, Q4/Black Friday, ограничения данных и SKU.
19. **18 · Что делать бизнесу** — прогноз на уровне категорий, event calendar, high-ticket planning.
20. **Спасибо / вопросы** — финальный Q&A-слайд.

## Текущая логика хода

1. Ввести задачу и ограничения данных.
2. Обосновать выбор 2017 года.
3. Показать методологию измерения сезонности.
4. Показать общий спрос.
5. Перейти к различиям между категориями.
6. Отдельно разобрать крупные покупки.
7. Сверить выводы через гипотезы и ML.
8. Закончить прогнозным и операционным выводом для бизнеса.

## Замечание для перестройки

Сейчас вопросы преподавателя не вынесены в начало презентации явно. При сильной переработке стоит построить ход вокруг четырёх вопросов из `assignment-questions.md`, чтобы каждый крупный блок закрывал конкретный вопрос задания.
